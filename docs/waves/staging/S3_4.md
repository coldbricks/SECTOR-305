## Fix

`packages/core/tests/scenario_json_load.test.ts` is a real consumer:

1. **`loadScenarioJson(relPath)`** — `readFileSync` + `JSON.parse` from repo-root paths; hard-fails if file missing.
2. **Field asserts** on both checkride and watch files (id, kind, packId, seed, title, tags, passConditions, authoring extras).
3. **`parseSharedScenarioSpine`** — maps shared fields (plus `expectedHardFailsOnFailSession` → `expectedHardFails`) through **`ScenarioSchema.parse`** so the JSON spine is Zod-validated, not only loosely keyed.
4. Watch path also asserts `durationMs`, `minCfs`, `minUnits`, `implementation`, and that the implementation module exists on disk.

Authoring-only extras (`implementation`, `minCfs`, `durationMs`, `expectedHardFailsOnFailSession`, `passConditions.note`) remain outside the strict Zod spine; they are asserted directly from the raw document.
